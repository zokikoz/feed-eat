&lt;h3 id=&quot;better-handling-of-negative-elements-in-enum&quot;&gt;&lt;a href=&quot;https://github.com/rails/rails/pull/40679&quot;&gt;Better handling of negative elements in enum&lt;/a&gt;&lt;/h3&gt;

&lt;h3 id=&quot;do-not-use-submit_tag-auto-disabling-when-disable_with-is-set-to-false&quot;&gt;&lt;a href=&quot;https://github.com/rails/rails/pull/40168&quot;&gt;Do not use submit_tag auto-disabling when disable_with is set to false&lt;/a&gt;&lt;/h3&gt;

&lt;p&gt;If we have &lt;em&gt;data: { disable_with: false }&lt;/em&gt; then auto-disabling is turned off, but if we set &lt;em&gt;automatically_disable_submit_tag&lt;/em&gt; to false it changes the behaviour of &lt;em&gt;disable_with&lt;/em&gt; in an unexpected way,¬† so explicit usage of &lt;em&gt;disable_with: false&lt;/em&gt; starts to enable auto-disabling with false as the value of a disabled button.&lt;br /&gt;
With this change auto-disabling is turned off if we have explicit &lt;em&gt;disable_with: false&lt;/em&gt; no matter what is set in &lt;em&gt;automatically_disable_submit_tag&lt;/em&gt;.&lt;/p&gt;

&lt;h3 id=&quot;ignore-strict-loading-violations-on-instances-loaded-through-fixtures&quot;&gt;&lt;a href=&quot;https://github.com/rails/rails/pull/40792&quot;&gt;Ignore strict loading violations on instances loaded through fixtures&lt;/a&gt;&lt;/h3&gt;

&lt;p&gt;Rails 6.1 added _strict_loading_by_default¬†_but that causes issues with the fixtures as they are lazy loaded. To get around it, this change ignores the strict loading flag when the fixtures are loaded.&lt;/p&gt;
